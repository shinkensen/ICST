TABLE OF TYPES

Time complexities Type Explanations
O(n) Linear Time to complete program scales linearly as n increases. Example: Looping through an array of n elements once.
O(1) Constant Runs the same amount of time no matter how large n is Example: Math operations, accessing elements in an array with an index.
O(n log n) Linearithmic Slightly slower than logarithmic time but faster than quadratic time. Usually a linear time step combined with a logarithmic time step. Example: Looping through a range of values, performing binary search on an array to search for that selected value, Merge Sort, Quick Sort
O(log n) Logarithmic Time grows logarithmically, increasing 1 step each time N doubles. Can be thought of as cutting the input size in half each step. Example: Binary search
O(nÂ²) Quadratic As N grows, runtime grows proportionally to the square of N. Example: Nested for loop to search for a given value in a 2D array. Checking all pairs in an array.
O(2^n) Exponential Time grows exponentially as N increases. Example: Looping through all subsets of a set with size N:
